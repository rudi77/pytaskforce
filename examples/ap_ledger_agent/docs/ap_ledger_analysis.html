Frontend

Telegram Bot (Long-Polling). Userin schickt Fotos, PDFs, Text. Agent antwortet mit Bestaetigungen.

Agent

Taskforce ReAct-Loop mit GPT-5.4-mini. Vision fuer Belegextraktion. 3 Tools: powershell, file_read, ask_user.

Persistenz

SQLite mit WAL-Mode. Atomare Transaktionen. Beleg-Archiv unter belege/. Audit-Trail.

Status-Lifecycle

  • Invoice: draft -> validated -> posted | rejected
  • Journal: draft -> posted | reversed
  • Kein DELETE — nur Status-Transitions

Transaktion

  • _use_conn() Context Manager
  • Shared Connection fuer alle Ops
  • BEGIN / COMMIT / ROLLBACK
  • Backward-compatible (conn=None)

Fehler-Output

  • Top-Level try/except in main()
  • Audit-Log bei Fehlschlag
  • Konsistenzcheck on-demand

Konfiguration

Eine Zeile in ap_ledger_agent.yaml: country: DE (oder AT). Steuert: Seed-Daten, Tax Codes, Compliance-Regeln, Monatsbezeichnungen. DB wird beim ersten Start automatisch mit den richtigen Daten initialisiert.

DATEV-Export (Groesste Luecke)

Aktueller CSV-Export

datum,lieferant,kategorie,netto,ust,brutto,steuersatz
2026-01-24,Tageslosung,Bareinnahmen,130.00,26.00,156.00,20% USt

Einfache CSV-Datei. Kein standardisiertes Format. Steuerberater muss manuell importieren.

Benoetigter DATEV-Export

  • Format: DATEV ASCII (CSV mit Header-Record)
  • Encoding: Windows-1252
  • Felder: Umsatz, Soll/Haben-Kz, BU-Schluessel, Gegenkonto, Belegdatum, Belegfeld1/2, Buchungstext, USt-ID
  • Konten: SKR03 oder SKR04 Mapping
  • Tax-Mapping: DATEV-Steuerschluessel (z.B. 9 = 19% VorSt)
  • Header: Berater-Nr, Mandanten-Nr, WJ-Beginn, Sachkontenlnge

Implementierungs-Aufwand

Neues Script export_datev.py das die DATEV-Buchungsstapel-Spezifikation implementiert. Benoetigt: Konten-Mapping-Tabelle (internes Konto -> SKR03/04), DATEV-Steuerschluessel-Mapping, Header-Konfiguration (Berater-Nr, Mandant). Geschaetzter Aufwand: 1-2 Tage.

Zusammenfassung

Staerken

  • Einfach: 1 Foto -> 1 Buchung (unter 10 Sekunden)
  • Atomar: Alles oder nichts, keine Inkonsistenzen
  • Praezise: Decimal-Arithmetik, keine Rundungsfehler
  • Nachvollziehbar: Audit-Log, Beleg-Archiv, Storno statt Delete
  • Flexibel: AT/DE-Konfiguration, erweiterbare Kategorien
  • Leichtgewichtig: SQLite, Python-Scripts, keine externen Services

Offene Punkte

  • DATEV-Export fehlt (wichtigste Luecke)
  • Kein kryptographischer Audit-Trail
  • Keine Verschluesselung at rest
  • Keine automatischen Backups
  • USt-VA Report fehlt
  • Keine Mehrbenutzerfaehigkeit

Fazit

Fuer eine Kleinunternehmerin mit EUeR ist der AP-Ledger Agent funktional vollstaendig. Die groessten Luecken (DATEV, Audit-Chain) betreffen die Schnittstelle zum Steuerberater und die Prueffestigkeit bei einer Betriebspruefung — beides mit ueberschaubarem Aufwand nachruestbar.
